                Looks like there is a new protocol version as specified in the wiki which does not use static keys anymore. This makes it even simpler and more denyable, but it puts more burdon on the initial handshake.                                 CodesInChaos • 9 months ago

                                I like using DH to authenticate, like it's used in djb's CurveCP or trevp's Noise.

                                But there is one serious limitation: All relevant keys need to use the same group. For example if Alice has a Curve25519 long term key, and Bob a P521 long term key they can't communicate with this protocol.

                                For my own protocols I didn't have a bit problem with that, since I use Curve25519/Ed25519 for everything. But I think many implementers prefer to support multiple kinds of key pairs. For example Noise supports P521 and Curve25519.

                                It is possible to avoid this by generating more ephemeral keys (one for confidentiality and one matching the partner's public key), but that adds a bit of complexity again.
